## 什么是搬瓦工套餐升级？

搬瓦工套餐升级，指的是在不重装系统、不更换 IP 的前提下，通过官方后台将现有 VPS 迁移到更高配置套餐的操作，系统会自动按剩余天数折算差价，只需补交费用差额即可。

简单说：花差价，换更高配置，数据和 IP 原封不动。

这个功能适用于搬瓦工主流常规套餐之间的横向升级，比如从 KVM 入门套餐升级到 CN2 GIA-E，或者从 CN2 GIA-E 小配置升到大配置。

流量跑完了，是最常见的触发点。但除了流量，以下几种情况同样值得考虑：

## 搬瓦工升级套餐详细步骤

整个流程分 4 步，全在浏览器里完成：

**第 1 步：登录后台，进入 My Services**

打开搬瓦工官网，点击右上角 "Client Area" 输入账号密码登录。登录后，点击顶部导航 `Services` → `My Services`，进入你的 VPS 实例列表页面。

**第 2 步：找到对应 VPS，选择 Upgrade**

在 Manage Services 页面，找到你想要升级的那台 VPS。点击该 VPS 行右侧的**齿轮图标 ⚙️**，在下拉菜单里选择 **"Upgrade"**（升级），进入升级操作页面。如果你有多台 VPS，注意核对 hostname 和 IP，别选错了。

**第 3 步：选择目标套餐**

页面会列出当前可供升级的所有套餐。根据你的需求选一个，同时选好付费周期（月付、季付、半年付、年付），费用越长折算下来越划算。如果有优惠码，这步顺手填上。选好后点击 **"Upgrade to this plan"**。

**第 4 步：确认差价，完成支付**

系统自动计算当前套餐剩余价值与新套餐之间的差额，生成账单。按页面提示完成支付，付款成功即升级生效，后台自动处理，通常几分钟内完成。

## 差价是怎么算的？

系统会自动帮你算，不需要你手动计算。但理解计算逻辑，有助于你判断现在升合不合算。

基本公式是：**差价 = (新套餐价格 - 旧套餐价格) ÷ 计费周期天数 × 当前套餐剩余天数 × 优惠码折扣**

举个例子：年付 $49.99 的 KVM 套餐，已经用了 100 天，想升到年付 $169.99 的 CN2 GIA-E 套餐，并使用 6.77% 优惠码：

差价约为 ($169.99 - $49.99) × (365 - 100) ÷ 365 × 0.9323 ≈ $81.4

以系统显示的最终金额为准，上面只是理解思路用的。

## 升级后要做什么？

升级完成后，做这几步：

1. **手动重启 VPS**：进入 KiwiVM 控制面板 → Main Controls → 点击 "Restart"，让新分配的硬件资源正确加载。
2. **验证配置**：SSH 登录服务器后，用 `free -m` 检查内存，`df -h` 检查磁盘，确认新配置已生效。
3. **重新评估机房选择**：如果是从 KVM 升到 CN2 GIA-E，现在可以解锁高端机房了。进 KiwiVM → Migrate → 选择 DC6 CN2 GIA-E、DC9 CN2 GIA 或日本软银 JPOS_1，国内访问速度会有明显改善。
4. **打开自动备份**：CN2 GIA-E 及以上套餐支持免费自动备份，进 KiwiVM → Backup 开启即可。
5. **创建新快照**：确认系统稳定运行后，再存一个快照作为当前健康状态的还原点。

## 升级套餐 vs 直接换购新套餐，哪个更合适？

大多数情况下，**升级套餐更划算，也更省事**。

原因很简单：升级只需补差价，IP 不变，数据不动，服务不中断。重新购买新套餐则需要重新配置系统环境、重新迁移所有数据，还要更新所有与 IP 绑定的服务——时间成本远高于省下的那点钱。

唯一值得考虑重新买的场景：你的当前套餐系列和目标套餐系列之间不支持直接升级（比如 KVM 套餐不能直接跨系列升到限量版套餐），这种情况下才需要买新的。具体能不能升，进升级页面看一眼可选项就清楚了。
